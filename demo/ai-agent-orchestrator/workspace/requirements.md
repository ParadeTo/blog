# 需求文档：员工休假记录管理系统

## 项目概述

开发一个员工休假记录管理系统的前端表单页面，支持对休假记录进行增删改查（CRUD）操作。

## 数据字段

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 记录唯一标识（UUID，系统生成） |
| employee_id | string | 员工工号（必填） |
| employee_name | string | 员工姓名（必填） |
| leave_type | enum | 休假类型：年假/病假/事假/婚假/产假/陪产假/丧假 |
| start_date | date | 开始日期（必填） |
| end_date | date | 结束日期（必填，须 >= start_date） |
| days | number | 休假天数（根据 start_date/end_date 自动计算） |
| apply_time | datetime | 申请时间（系统自动填入） |
| status | enum | 状态：待审批 / 已批准 / 已拒绝 |
| approver | string | 审批人姓名 |
| reason | string | 申请理由（必填，最多 200 字） |

## 功能需求

### 列表页
- 展示所有休假记录，支持按员工姓名、休假类型、状态筛选
- 每行显示：员工姓名、休假类型、开始日期、结束日期、天数、状态、操作按钮

### 新建/编辑表单
- 支持新建休假记录
- 支持编辑现有记录（状态为"待审批"时可编辑）
- 表单验证：必填项、日期合法性、天数自动计算
- 提交成功后刷新列表

### 删除
- 只允许删除"待审批"状态的记录
- 删除前弹窗确认

### 状态变更（审批）
- 审批人可将"待审批"记录改为"已批准"或"已拒绝"

## 技术要求

- 前端：原生 HTML + JavaScript（不使用框架，保持简单易懂）
- 后端：Node.js + Express（接口简单，聚焦演示）
- 数据存储：内存存储（Map），无需数据库
- 接口风格：RESTful JSON API
- 环境：本地开发，无需部署

## 验收标准

1. 前端页面可在浏览器正常打开，列表/新建/编辑/删除功能可用
2. 后端接口符合 api_spec.md 定义
3. 所有接口有对应的测试，并能通过
4. 交付物包含完整的项目结构说明
