# 项目交付报告

**项目名称：** 员工休假记录管理系统
**技术栈：** Node.js 18+ + Express + 原生 HTML/JS + 内存 Map 存储
**报告日期：** 2025-07-10

---

## 1. 需求摘要

本系统为企业内部员工休假记录管理平台，提供完整的休假申请、查询、审批与管理功能，主要功能如下：

- **休假申请**：员工可填写姓名、休假类型、开始/结束日期、申请原因，提交休假申请。
- **记录列表**：以列表形式展示所有休假记录，支持按员工姓名、休假类型、审批状态进行筛选查询。
- **记录编辑**：对已提交的休假申请进行修改（仅限待审批状态）。
- **记录删除**：删除指定的休假记录。
- **审批管理**：管理员可对休假申请进行审批操作，支持"批准"与"拒绝"两种结果，并可填写审批意见。
- **数据校验**：前后端均对必填字段、日期合法性、枚举值等进行校验，保障数据完整性。

系统采用前后端一体化部署方式，前端页面通过 Express 静态文件托管提供访问，数据存储于服务端内存中，适用于轻量级演示与原型验证场景。

## 2. 快速启动指南

### 环境要求

| 环境 | 版本要求 |
|------|----------|
| Node.js | 18.0.0 及以上 |
| npm | 随 Node.js 附带 |

### 启动步骤

```bash
# 第一步：安装项目依赖
npm install

# 第二步：启动后端服务
npm start
# 或使用以下等效命令：
node server/app.js

# 第三步：访问前端页面
# 打开浏览器，访问以下地址：
# http://localhost:3000

# 第四步：运行接口测试（请确保后端服务已启动后再执行）
node --test tests/leaves.test.js
```

### 页面入口说明

| 页面 | 访问地址 | 说明 |
|------|----------|------|
| 休假记录列表页 | http://localhost:3000 | 主页，含筛选与审批功能 |
| 新建休假申请页 | http://localhost:3000/form.html | 提交新的休假申请 |
| 编辑休假申请页 | http://localhost:3000/form.html?id={id} | 编辑指定记录（携带 id 参数） |

## 4. 接口清单

基础路径：`/api/leaves`

| # | 方法 | 路径 | 描述 |
|---|------|------|------|
| 1 | `GET` | `/api/leaves` | 获取休假记录列表，支持按姓名、类型、状态筛选（Query 参数） |
| 2 | `POST` | `/api/leaves` | 新建休假申请，返回创建成功的记录（含自动生成的 UUID） |
| 3 | `GET` | `/api/leaves/:id` | 根据 ID 获取单条休假记录详情 |
| 4 | `PUT` | `/api/leaves/:id` | 根据 ID 更新休假记录信息（仅限待审批状态） |
| 5 | `DELETE` | `/api/leaves/:id` | 根据 ID 删除指定休假记录 |
| 6 | `PATCH` | `/api/leaves/:id/approve` | 对指定记录进行审批操作（批准 / 拒绝），可附带审批意见 |

> 详细的请求参数、响应体结构及错误码定义，请参阅 `design/api_spec.md`。

## 6. 已修复问题记录

| # | 问题文件 | 问题描述 | 修复方案 | 状态 |
|---|----------|----------|----------|------|
| 1 | `public/js/list.js` | UUID 格式的记录 `id` 在 HTML `onclick` 属性中拼接时未加引号，导致 JS 语法错误，点击操作按钮时功能失效 | 在拼接 `onclick` 属性字符串时，为 `id` 变量两侧添加单引号，确保 UUID 作为字符串参数正确传递 | ✅ 已修复 |
| 2 | `public/js/api.js` | `fetch` 响应状态判断逻辑仅将 `200` 视为成功，导致后端返回 `201 Created` 时被误判为请求失败，新建记录后前端报错 | 将成功状态判断条件修改为 `response.ok`（覆盖 200–299 全部成功状态码），兼容 `201` 等非 200 成功响应 | ✅ 已修复 |

## 7. 假设与说明

| # | 类别 | 说明 |
|---|------|------|
| 1 | **部署方式** | 前端页面通过 Express 静态文件托管（`express.static`）对外提供访问，无需单独部署前端服务，统一入口为 `http://localhost:3000` |
| 2 | **数据持久化** | 数据存储于服务端内存 Map 中，服务重启后数据将全部清空，此行为符合当前需求定位（原型演示），如需持久化可后续接入数据库 |
| 3 | **测试前置条件** | 接口测试（`tests/leaves.test.js`）依赖正在运行的后端服务，执行测试前须确保已通过 `npm start` 启动服务 |
| 4 | **跨域处理** | 后端已引入 `cors` 中间件，支持跨域请求，便于在不同端口或工具（如 Postman）中直接调用接口 |
| 5 | **模块规范** | 项目 `package.json` 中设置 `"type": "module"`，全部 JS 文件均使用 ES Module（`import/export`）语法 |
| 6 | **ID 生成** | 每条休假记录的唯一标识由 `uuid` 库自动生成（v4 格式），无需手动指定 |
